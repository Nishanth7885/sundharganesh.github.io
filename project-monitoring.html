Project Overview

A hands-on learning project where I successfully demonstrated the implementation of a complete monitoring stack using Prometheus, Grafana, and node_exporter.

This project was undertaken as part of my DevOps learning journey to gain practical experience with industry-standard monitoring tools. I installed and configured all three components on Ubuntu Linux servers, learning about time-series databases, metric collection, and data visualization in the process.

Through this hands-on experience, I learned how to set up monitoring infrastructure from scratch, troubleshoot configuration issues, and understand how modern DevOps teams maintain visibility into their systems. All installation steps and configurations are documented on GitHub for future reference and to demonstrate my learning process.

Challenges & Learning Points

01

Prometheus Configuration Syntax

Challenge:

Service failed to start due to YAML syntax errors and incorrect indentation in prometheus.yml configuration file.

What I Learned:

Used promtool check config prometheus.yml to validate configuration before starting service. Learned the importance of proper YAML indentation and syntax validation in DevOps workflows.

02

Service Discovery & Target Configuration

Challenge:

Prometheus targets showed "DOWN" status. Could not scrape metrics from node_exporter endpoints.

What I Learned:

Understood the importance of network connectivity, firewall rules, and service status checks. Learned to use curl http://localhost:9100/metrics to test endpoints and systemctl status to verify service health.

03

systemd Service Creation

Challenge:

Services would not persist after reboot. Had to manually start services each time.

What I Learned:

Created proper systemd unit files and learned to use systemctl enable for auto-start on boot. Understood Linux service management and daemon configuration.
