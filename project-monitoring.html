Infrastructure Monitoring Stack

Prometheus + Grafana + node_exporter

2023 - 2024 DevOps Project Production Environment

Project Overview

Designed and deployed a complete infrastructure monitoring solution to track system health across multiple Ubuntu Linux servers in a hospital environment.

This project addressed the critical need for real-time visibility into server performance metrics. I built custom Grafana dashboards that visualize CPU, memory, disk, and network utilization, enabling proactive infrastructure management and rapid troubleshooting of performance issues.

The monitoring stack runs 24/7, collecting metrics from 10+ production servers and providing instant alerts when system thresholds are exceeded. All configuration and implementation steps were documented on GitHub for reproducibility and knowledge sharing.

Technical Implementation

Infrastructure Setup

  • Ubuntu Linux 20.04 LTS servers
  • Prometheus 2.x installed from binary
  • Grafana 9.x installed via APT repository
  • node_exporter deployed on all monitored hosts

Configuration Management

  • YAML-based Prometheus configuration files
  • systemd service units for process management
  • Scrape intervals configured to 15s
  • Static target discovery for servers

Service Architecture

  • Prometheus: Port 9090 (metrics & UI)
  • Grafana: Port 3000 (dashboards)
  • node_exporter: Port 9100 (metrics endpoint)
  • Firewall rules configured for service access

Data & Storage

  • Time-series database with 30-day retention
  • Metric storage in /var/lib/prometheus
  • Grafana dashboards stored in SQLite
  • Configuration backup to GitHub

Challenges & Solutions

01

Prometheus Service Startup Failure

Challenge:

Prometheus failed to start with YAML syntax errors in configuration file. Service logs showed indentation and parsing issues.

Solution:

Used promtool check config prometheus.yml to validate configuration syntax. Corrected indentation issues and verified scrape target formatting. Implemented configuration validation as part of deployment process.

02

node_exporter Metrics Not Appearing

Challenge:

Prometheus targets showed as "DOWN" status. node_exporter metrics endpoint was not accessible from Prometheus server.

Solution:

Verified node_exporter service status with systemctl status node_exporter. Checked firewall rules and opened port 9100. Tested connectivity with curl http://target:9100/metrics to confirm accessibility.

03

Grafana Dashboard Data Source Issues

Challenge:

Grafana panels showed "No data" errors despite Prometheus collecting metrics successfully.

Solution:

Configured Prometheus as data source in Grafana with correct URL (http://localhost:9090). Tested connection and verified PromQL queries. Adjusted time ranges and metric names to match actual data available.
